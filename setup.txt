 Phase 5 
 eval_one:
  - Runs the gold SQL first to get the reference rows
  - Calls the agent with eval_run: baseline tag (Langfuse will capture this)
  - Extracts SQL from only generate_sql and revise history entries (skips verify entries, which have no sql key)
  - Runs and scores each iteration's SQL independently against gold rows — so you get per-iteration accuracy, not just
  final
  - Returns per_iter_correct: {0: bool, 1: bool, ...} keyed by iteration index 
 To run:
  # agent server must be running
  uv run python evals/run_eval.py --out results/eval_baseline.json

  The output JSON will have summary.per_iteration_accuracy — if iter_0 == iter_2, the verify→revise loop is not helping. If
  iter_2 > iter_0, it's earning its keep.
